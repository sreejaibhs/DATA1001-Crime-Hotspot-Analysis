#  Crime Hotspot Analysis using R (DATA1001 Project)

## Project Overview
This project was completed as part of the DATA1001 unit. It analyzes crime data from New York City to identify spatial (borough-based) and temporal (time-based) crime hotspots. The goal is to provide data-driven insights to improve police deployment strategies.

## Key Objectives
- Identify boroughs with the highest crime rates
- Analyze hourly crime patterns
- Detect peak crime hours
- Provide actionable recommendations for law enforcement

## Tools & Technologies
- R
- ggplot2
- dplyr
- R Markdown

## Data Source
NYC Open Data – NYPD Complaint Data Historic Dataset

## Key Findings
- Brooklyn and Manhattan show the highest crime rates
- Crime peaks between **2 PM – 9 PM**
- Significant variation exists across boroughs

## Statistical Analysis
- Applied **Kruskal-Wallis Test** due to non-normal data
- Found statistically significant differences in hourly crime distribution across boroughs

## Recommendation
Increase police patrols in Brooklyn and Manhattan during afternoon and evening hours (2 PM – 9 PM). Maintain standard patrol levels in Staten Island and Queens with targeted night monitoring.

## Limitations
- Based only on reported crimes
- Limited time period
- Possible data inconsistencies
